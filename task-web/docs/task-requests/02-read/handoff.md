# 02 Read Handoff

## Conversation Context

- 사용자는 read-side 결론만 보는 것이 아니라, 왜 이 방향으로 갔는지에 대한 짧은 대화 맥락도 다음 에이전트에게 남기길 원한다.
- 기록은 full transcript가 아니라 사용자의 현재 목표, 명시한 제약, 대화 중 잠긴 결정만 담아야 한다.
- 이 track도 기술 handoff 앞에 사람용 대화 맥락을 두고, 같은 내용을 `status.json.conversation_context`로도 남긴다.
- read track에서는 이 기록 규칙을 유지하되 exact runtime read-path 우선 원칙과 lossy fallback 금지 원칙은 그대로 보존해야 한다.
- 현재 paragraph-style 범위는 `Read paragraph style (alignment, line spacing)`에만 한정한다. current selection/caret formatting API를 새 capability로 승격하지 않는다.
